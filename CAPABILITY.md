# Workflow Repair Desk — Capability Summary

## Focus

Bounded automation repair and implementation for small teams using n8n, JavaScript/Node, webhooks, APIs, spreadsheets, CRMs, browser tools, and AI-assisted workflows.

## Demonstrated in this repository

- deterministic lead-input normalization and validation;
- scoring and routing with visible rejection states;
- n8n Code-node JavaScript parity-tested against a companion Node implementation across three fixtures;
- safe synthetic test data and automated CI configuration;
- acceptance-test-first OpenClaw VPS hardening design;
- systemd, SSH/firewall, rollback, secrets, logging, and verification planning;
- human review for ambiguous or consequential actions.

## Delivery method

1. Define one business outcome and its acceptance test.
2. Confirm access boundaries, dependencies, failure behavior, and rollback.
3. Build or repair the smallest reliable workflow that meets the test.
4. Deliver test evidence, exportable artifacts, known gaps, and handoff notes.

## Commercial starting point

- Bounded launch pilot: starts at USD 250 after scope and acceptance criteria are agreed.
- Larger production work: quoted after dependency review or a successful pilot.
- Ongoing monitoring/support: defined separately with response windows and spending caps.

## Evidence boundaries

The repository contains self-directed demonstration assets. It is not presented as paid client work, a completed client deployment, a testimonial, or evidence of financial outcomes. Live-client access and production actions remain client-controlled and approval-gated.
